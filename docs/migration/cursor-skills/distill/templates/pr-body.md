# PR body template (from cursor-skills read-pr-body-sections)

Typical sections: **TL;DR** → **What changed** (nested bullets) → tradeoffs / risks.

- Do not add a standing "Linked issues" section by default.
- At most one trailing line when keyword path confirmed: `Refs #N` or `Fixes #N`.
- Merge happens in GitHub UI only — never via CLI.
