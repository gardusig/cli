# Issue body template (from cursor-skills)

Use issue body for stable scope + acceptance criteria. After create, prefer append-only
`## [cli] plan|question|warning|outcome` comments instead of editing the body.

Suggested sections:
- **Goal** — one line
- **Acceptance** — checklist
- **Depends** — `#N` for topological ordering within epic
